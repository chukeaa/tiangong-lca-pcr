# Feedback Loop

Use feedback when Agent work discovers PCR gaps or improved evidence.

Feedback should include:

- PCR id and version
- feedback type
- affected section, process_id, and flow role when applicable
- current PCR excerpt
- proposed change
- evidence sources
- impact on process or lifecyclemodel construction
- confidence

Submit feedback through the GitHub issue templates or by drafting issue-ready Markdown with `tiangong-pcr feedback draft`.
