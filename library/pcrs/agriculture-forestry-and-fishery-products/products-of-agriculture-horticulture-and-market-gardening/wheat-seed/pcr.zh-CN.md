# 小麦种子

## 1. 状态与用途

本 PCR 是用于评估 TianGong PCR Markdown 结构的样例草案。它可以用于测试人工和 Agent 的 PCR 编写、检索和建模工作流；其中的数值范围和建模选择在正式使用前仍需要农业与 LCA 专家复核。

本 PCR 适用于构建用于播种的小麦种子生命周期模型或单元过程，包括认证种子、原种、基础种或同等用途的种子。若研究对象是食用、饲用、淀粉、乙醇或普通商品交易用小麦籽粒，不应直接使用本 PCR，除非研究边界已明确排除种子专属处理过程并重新分类产品。

## 2. 产品类别身份

- Canonical PCR id: `pcr.agriculture-forestry-and-fishery-products.products-of-agriculture-horticulture-and-market-gardening.wheat-seed`
- 外部分类别入口: CPC 3.0 `01111`, `Wheat, seed`
- 产品类型: 农产品
- 典型生产系统: 田间繁种加采后种子加工
- 典型市场状态: 清选、分级、包装、可选包衣或药剂处理后，在农场门、种子加工厂门或区域配送点交付的小麦种子

PCR 身份不由 CPC 决定。CPC 只是入口和映射层；本 PCR 才是建模规则的 canonical 记录。

## 3. 参考流

优先使用的参考流是：

```text
1 kg wheat seed for sowing, cleaned and graded, at seed plant gate
```

可接受的变体：

| 参考流 | 优先单位 | 必填限定信息 |
| --- | --- | --- |
| 未处理小麦播种种子 | kg | 种子等级、含水率基准、发芽率、净度、地区、交付边界 |
| 已处理小麦播种种子 | kg | 药剂或包衣有效成分、施用量、种子等级、含水率基准、交付边界 |
| 袋装认证小麦种子 | kg | 包装材料、包装规格、药剂处理状态、储存时间 |
| 散装小麦种子送达农场 | kg 或 tonne | 运输距离、运输方式、药剂处理状态 |

除非研究目标直接关注农艺表现，否则不建议用粒数作为唯一参考流。如使用粒数，必须同时提供千粒重或其他可换算为质量的参数。

## 4. 流属性与单位约定

主要流属性使用 `mass`。常用单位为 `kg` 和 `tonne`。

可获得时应声明以下质量属性：

| 属性 | 常用单位 | 常见筛查范围 |
| --- | --- | --- |
| 含水率 | 湿基百分比 | 10-14 percent |
| 物理净度 | 质量百分比 | 95-99.9 percent |
| 发芽率 | 种子百分比 | 85-98 percent |
| 千粒重 | g/1000 seeds | 25-60 g |
| 种子处理载量 | g active ingredient/kg seed | 0-10 g/kg |

这些范围只用于筛查和建模审查，不是合格判定标准。存在本地种子认证标准时，应优先使用本地标准。

## 5. 系统边界

默认边界为 cradle-to-gate 小麦种子生产：

1. 上游投入品生产与运输。
2. 田间繁种：整地、播种、施肥、灌溉、植保、收获。
3. 采后处理：干燥、清选、分级、种子检测、药剂处理、包装和储存。
4. 当参考流为送达种子时，包含至声明交付边界的运输。

除非生命周期模型明确研究“种子作为小麦种植投入”的下游使用阶段，否则不包含使用该种子生产粮食小麦的过程。默认不包含资本品；若研究目标或显著性判断要求，应单独纳入。若影响方法或研究目标包含土地使用，应纳入土地占用和土地转化。

## 6. 常见过程拆分

数据允许时，生命周期模型应拆分为以下过程：

| 过程 | 角色 | 建模说明 |
| --- | --- | --- |
| 原种或基础种供应 | 上游产品输入 | 作为繁种田的种子投入建模 |
| 整地与播种 | 前景过程 | 包含播种量、耕作、机械燃料和田间损失 |
| 养分管理 | 前景过程 | 包含矿质肥、有机肥、石灰和养分相关排放 |
| 植保管理 | 前景过程 | 包含除草剂、杀菌剂、杀虫剂；必要时包含种子田检验 |
| 灌溉 | 条件性前景过程 | 灌溉系统应包含抽水能源和取水 |
| 收获与田间干燥 | 前景过程 | 包含联合收割燃料、收获含水率和田间损失 |
| 种子干燥与清选 | 前景过程 | 包含干燥能源、清选电力、不合格物和粉尘 |
| 种子处理与包装 | 条件性前景过程 | 包含有效成分、包衣材料、包装袋、托盘和废弃物 |
| 储存与门到门运输 | 条件性前景过程 | 包含储存用电、损失和已声明运输段 |

## 8. 分配与副产品处理

只有存在可证明的物理关系时才使用物理划分。否则按以下顺序处理：

1. 通过过程细分避免分配，例如拆分种子生产、粮用副产物、秸秆处理和筛下物处理。
2. 只有在被替代产品和市场有证据时，才使用替代法。
3. 当种子、类粮食筛下物和秸秆均为市场产品且价格数据可靠时，使用经济分配。
4. 质量分配仅作为兜底方法，并应标记为需要复核。

不合格种子、筛下物和秸秆不能在模型中消失。它们必须有明确去向：产品输出、废物处理、还田、露天焚烧、翻埋、饲料利用或其他有证据的去向。

## 9. 数据质量要求

最低前景数据要求：

- 地理范围和生产年份，或多年平均
- 田间产量和清选后种子产量
- 播种量和种子等级
- 肥料类型和施用量
- 灌溉状态和水源
- 田间燃料用量或作业清单
- 干燥、清选和包装能源
- 种子处理状态和有效成分
- 副产品和废弃物去向

优先数据质量：

- 天气敏感地区使用至少三年田间产量平均值
- 种子清选产率和不合格率使用一手数据
- 氮肥排放因子使用区域化模型
- 当加工能源占比较高时使用供应商或地点电力结构
- 对收获和清选后种子的含水率基准进行显式声明

## 10. Agent 建模指令

当 Agent 基于本 PCR 构建 `lifecyclemodel` 时：

1. 创建名为 `wheat seed production and conditioning` 的前景过程。
2. 将定量参考设置为已声明的参考流。
3. 数据允许时，将田间生产、清选加工、药剂处理、包装和门到门运输拆分为独立过程。
4. 对所有流标记 `product`、`waste` 或 `elementary`；不要仅凭流名称推断基本流。
5. 对所有来自筛查范围的假设保留不确定性和来源说明。
6. 若必填值未知，插入 review placeholder，不要虚构精确值。

## 11. 校验清单

发布使用本 PCR 的 process 或 lifecyclemodel 前，应检查：

- 参考流包含质量、交付边界、种子处理状态和种子质量限定
- 模型区分种子产品和普通商品粮
- 肥料、燃料、灌溉、加工能源、包装和种子处理投入均已处理
- 范围内已考虑 N2O、氨、硝酸盐、化石 CO2、取水和土地占用
- 筛下物、不合格种子、秸秆和包装废弃物有明确去向
- 分配方法已声明并给出理由
- 所有超出筛查范围的值都有来源说明或 review flag
